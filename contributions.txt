Contributions of the work include:

- A novel method of scoring the semantic alignment of knowledge graph instances against ontology classes, based on exact or fuzzy matching to OWL DL restriction-based constraints
- An algorithm for generating these scores at scale, compatible with SIMD execution on GPUs and analyzed for time and memory complexity
- An evaluation of the algorithm, comparing the results and performance of a Python implementation against equivalent SPARQL queries on Fuseki and reasoner invocations on Pellet
- A preliminary exploration of use cases for the work, accompanied by a publically accessible Python module supporting applications in knowledge graph construction, consistency checking, SHACL constraint checking, and inference production compatible with OWL EL and the majority of OWL RL 


The semantics of the constraint DAG are as follows:
- Broadly: 
  - According to known evidence, to what degree does a KG instance match necessary conditions of membership in a given class?
  - Given a set of KG instances and a set of ontology classes, to what degree does each instance correspond to each class?
  - Given a set of KG instances and a set of ontology classes, to what degree does evidence support that each instance meets the necessary criteria for membership in each class?
- Answers the question:
   - Is it safe to assume instance n meets all necessary criteria for membership in class C?
   - Can I classify instance n as class C without introducing inconsistency?
    - For OWL EL: yes
- Formally: 
  - Assume that the KG is already consistent. Given an instance n in the KG and a class C in the KG, a score of s(n,C) = 1.0 indicates that if instance n was classified as class C, the graph would remain consistent.
    - CWA: No local inconsistencies would be identified by an OWL DL 2 reasoner based on the current KG.
      - Possibly viable with OWL DL or OWL RL, maybe only OWL EL or positive monotone increasing subset
      - Not likely to apply if ALL are added -- that wouldn't work I don't think
    - OWA: If new information was added to the KG, and that information is consistent with what was already in the KG, the assertion that n is a member of C would continue to be consistent.
      - OWL EL: only if OWL EL assertions are allowed I think

- cardinality: written for >=k and <=k for simplicity, but would be implemented for <k and >k
  - given a cardinality restriction >=k on S P O: 
    - for all triples t_i = (s,?p,?o) in G:
      - compute score(t_i) = score(?p,P) * score(?o,O)
    - get the set of top k scores for each triple: T_k = {t_0,...,t_(k-1)}
    - for strict cardinality: result is min(T_k) 
      - note: existential restriction is essentially >=1, so it should be equivalent to that (so it should be computed with property score * object score, if there are property scores provided)
    - for fuzzy: result is sum(T_k)/k
      - fuzzy aggregation/intersection operator should match this; I think avg is close enough for now
  - <=k and ==k should match this
    - <=k is essentially NOT(>=(k+1)): 
      - strict <=k: 1.0 - min(T_(k+1))
      - fuzzy <=k: 1.0 - sum(T_(k+1))/(k+1)
    - ==k is essentially AND(<=k,>=k): 
      - strict ==k: min(min(T_k), 1.0 - min(T_(k+1)))
      - fuzzy ==k: avg(sum(T_k)/k, 1.0 -(sum(T_(k+1)))/(k+1))
  - For fuzzy cardinality, the delta parameter would control the relative spread, if implemented; that is likely a stretch goal for no
